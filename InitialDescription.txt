Learning Objective RecapThe primary goal of this exercise is to understand how an AI coding harness interfaces with features like Skills, Templates, Tools, and Model Context Protocol (MCP) Servers.To demonstrate this interaction practically, we will configure an AI harness to build a feature-complete, single-file HTML5 mobile-responsive application. The application will serve as a Cyclist Navigation & Power Telemetry Dashboard with the following capabilities:Core Navigation: Real-time mobile GPS tracking, automated map re-centering, route plotting between a start and end point using OpenStreetMap/Leaflet, and an on-screen route overview inset.Dynamic Rerouting: Automatic route recalculation if the rider deviates from the planned path.Advanced Telemetry: Live calculation of rider speed, total distance travelled, cumulative elevation gain, and real-time mechanical power output (\(W\)) based on rider/bike weight, speed, and slope gradient.Gradient Awareness: Color-coded route lines mapping upcoming slope steepness, dynamic countdowns of distance remaining for sustained climbs (\(>5\%\) gradient), and calculated minimum balance speed thresholds alongside the wattage required to sustain them.Step 1: Mapping the Project to an MCP & Tools HarnessTo build this project using an open-source agent harness like OpenCode or Pi, the agent relies on structured tools and protocol servers. Here is how the harness utilizes these components to build your app:[ Your AI Agent Harness ]

MCP Servers (Model Context Protocol): The harness connects to an mcp-server-filesystem to securely create, read, and write your project files. It also utilizes an mcp-server-fetch to query external routing endpoints (like the Open Source Routing Machine) to understand how the API response format maps to geometry.Tools: The agent executes atomic functions like write_file, view_code, and bash_execute (to run a local preview web server).Skills / Templates: The agent pulls pre-configured computational skills (such as physics formulas for rolling resistance and aerodynamic drag) to ensure the JavaScript telemetry code is mathematically sound.Step 2: The Physical & Mathematical FormulasTo ensure the JavaScript code calculates telemetry accurately, the underlying Computational Skill injected by the harness utilizes standard cycling physics equations.Mechanical Power EquationThe total power required (\(P_{\text{total}}\)) to propel a bicycle is the sum of the power needed to overcome gravity, aerodynamic drag, and rolling resistance:\(P_{\text{total}}=\frac{P_{\text{gravity}}+P_{\text{drag}}+P_{\text{rolling}}}{\eta }\)Where:\(P_{\text{gravity}} = m \cdot g \cdot \sin(\arctan(G)) \cdot v\)\(P_{\text{drag}} = 0.5 \cdot C_d \cdot A \cdot \rho \cdot v^3\)\(P_{\text{rolling}} = m \cdot g \cdot \cos(\arctan(G)) \cdot C_{rr} \cdot v\)\(m\) = Total mass (rider + bicycle in \(\text{kg}\))\(g\) = Acceleration due to gravity (\(9.81 \, \text{m/s}^2\))\(G\) = Gradient as a decimal (e.g., \(0.05\) for \(5\%\))\(v\) = Velocity relative to the ground (\(\text{m/s}\))\(C_d \cdot A\) = Aerodynamic drag coefficient area (assumed \(\approx 0.38\))\(\rho \) = Air density (assumed \(\approx 1.2 \, \text{kg/m}^3\))\(C_{rr}\) = Rolling resistance coefficient (assumed \(\approx 0.005\) for pavement)\(\eta \) = Drivetrain efficiency (assumed \(\approx 0.95\) for a clean chain)Minimum Balance Speed & Stability WattageTo maintain bicycle stability without stalling or weaving excessively on steep inclines, a baseline speed is required. A practical minimum speed threshold for stable bicycle tracking under load is approximately \(1.67 \, \text{m/s}\) (\(\approx 6 \, \text{km/h}\)).The stability wattage is calculated by running this minimum speed value through the total power equation for the current gradient.Step 3: Complete Single-Page HTML5 ImplementationThis single file contains all structural elements, CSS layout styling optimized for mobile screens, and the core Leaflet mapping engine script.
